元素居中与CSS盒模型

元素居中方法总结

一、水平居中

1.块元素水平居中：margin:0px auto;（任何当代浏览器上都有效，即使是IE6标准模式）
这种情况下块元素需要设定具体width才能达到左右居中的效果。
通过这条语句可以使块元素相对于其父元素水平居中，且自动排列于父元素上部。不适用于内联元素。
example1:

2、text-align:center外加内部元素display:inline:block
这种情况可以有效的让块内元素迅速居中，缺点是每个行内块元素之间会出现4px的间隙。通过改变书写格式或者调整margin值可以修改
具体方式参阅：This link
example2:

3、浮动实现水平居中（其实与负margin方式类似）
将包含内容的块儿通过左浮动后进行定位至正中；然后通过将块儿内的内容浮动起来向右定位50%（这个距离是块儿的一半）
同时如果对外层容器进行高度设置后，可使用同样的方法达到垂直居中。
example3:

4.负边界
原理：通过绝对定位把需要居中的元素定位在父级元素的中央位置，再通过负值margin移动元素一半的宽高达到目的，
此时父级元素与居中元素的中心重合在一起。这种方式不光能够达到水平居中，垂直居中也同样适用！
缺点：必须知道具体的宽度或者高度才能够反向margin
example4:

5、flex布局达到水平或者垂直居中的效果
这部分内容进行单独总结，这里应用最基础的flex布局可以达到水平或者垂直居中。此方法注意不同浏览器的兼容性，为CSS3最新方法布局。

6、width的fit-content属性与margin配合达到水平居中
与上面相同，这也是一个新的width属性，这个属性可以解决水平居中，且不用特意去设定具体的宽度。
这个方法最大的缺点是现在只能应用在chrome和firefox，其他浏览器暂时无法兼容。

二、垂直居中

1、单行内容垂直居中
通过设置line-height与height等高即可
注：图片不适用这种情况，主要为文字，此处不举例说明了。
注：当具有多行内容时，可以同时放入一个span里，然后将其设置为display:inline-block然后设置等高行即可。

2、模拟表格
将display设置为table-cell之后，由于表格元素默认居中所以设置vertical-align:middle后即可居中
注：同样不是直接运用于图片上，可将图片放入table-cell块儿内达到效果(且ie67不支持这种方式，需要运用上面介绍的负边界方式达到效果)

3、display:inline-block借助另外元素高度来垂直居中
通过设置另一个元素高度达到父元素高度100%，然后同时为inline-block的情况下设置vertical-align:middle;即可达成。
注：这种方法可以完成多张高度不确定的图片同时垂直居中。

补充：不仅span可以让其具有大小，同样可以使用很大的字体font-size或者一个空的图片来给图片高度达到垂直居中

总结：总结的很多水平居中的方式其实都可以应用于垂直居中，特别着重注意的就是负margin值这种情况，因为不仅可以解决水平垂直两项问题。同时，在垂直居中很多情况下使用的vertical-align:middle不能兼容的情况下都可以使用负margin这种手段达到居中的补充手法。
另外，对于多内容的垂直居中，重点要掌握的其实是display:table-cell属性的用法，还有就是可以通过空元素撑开父元素的高，来给其他需要剧中的元素一个高度的标尺（垂直居中第三种）。

CSS盒模型介绍

1、盒模型四个边界：
content：由该元素的宽高决定矩形大小，没有上右下左；
padding：填充，内边距，向内挤压元素的边界。
border：元素的边框，可以设置其宽度。
margin：元素外边距。

2、margin：

I、margin不是table类型的元素，以及 table 类型中table-caption,table和inline-table 这3类。例TD、TR、TH等，margin是不适用的。 并且，对于行内非替换元素（例如 SPAN），垂直方向的 margin 不起作用。

II、垂直方向上的不同元素的相邻的 margin 在某些情况下，会发生折叠的现象。

外边距折叠：
Collapsing margins，即外边距折叠，指的是毗邻的两个或多个外边距 (margin) 会合并成一个外边距。

（1）两个或多个毗邻的普通流中的块元素垂直方向上的 margin 会折叠：

原因：没有被非空内容、padding、border 或 clear 分隔开。
一个元素与它子元素的第一个子元素margin-top相邻；当一个元素height为auto时与其最后一个子元素margin-bottom相邻。
折叠后 margin 的计算：
I、符号相同时两相邻元素margin值取绝对值较大的值作为两者margin值。
II、符号不相同的时候取正值最大与负值最小进行直接相加。（相当于挤开正值那么宽，再拉近负值那么宽）
复杂实例：

根据有正有负时的计算规则，正值的最大值为 200px，负值中绝对值最大的是 -120px，所以，最终折叠后的 margin 应该是 200 + (-120) = 80px。

（2）浮动元素、inline-block 元素、绝对定位元素的 margin 不会和垂直方向上其他元素的 margin 折叠

（3）创建了块级格式化上下文的元素，不和它的子元素发生 margin 折叠（与第二条内容重合）

注：块级格式化上下文至少满足以下条件之一：
float 的值不为 none
position 的值不为 static 或 relative
display 的值为 table-cell、table-caption、inline-block、flex 或 inline-flex
overflow 的值不为 visiable

（4）元素自身的 margin-bottom 和 margin-top 相邻时也会折叠，即没有内容填充的情况下margin撑不开元素。

III、border：

1、border 包含3个部分，’border-width’，’border- color’，’border-style’，分别用来设置它的宽度，颜色和样式。
其中border-width 可用值有‘thin’，‘medium’，‘thick’，以及常用的数值带单位的宽度值。
其中border-style 默认值是none特性的值。可用值有：’none’，’hidden’，’dotted’，’dashed’，’solid’，’double’，’groove’，’ridge’，’inset’等。
